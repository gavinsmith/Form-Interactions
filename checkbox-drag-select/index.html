The behaviour is simple. Focus a field, nothing happens. The label is positioned behind the input, and appears just as a placeholder attribute would, or a pre-populated value. A common technique for a while now has been to pre-populate and clear on focus, but this is really silly. The placeholder attribute behaves the same way, although it is not meant to serve the same purpose semantically. Instead, the abel is updated with each keystroke - if there's content in the input, the label hides, and if the input becomes empty, the label shows itself again.
